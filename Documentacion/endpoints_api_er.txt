================================================================================
          DOCUMENTACIÓN DE ENDPOINTS - SISTEMA SIGRAT (SUPABASE / POSTGRESQL)
================================================================================
Módulo: Arquitectura Base RESTful
Descripción: Mapeo completo de las rutas de la API para el sistema SIGRAT.
Los payloads JSON y la estructura están diseñados estrictamente para coincidir 
con la base de datos alojada en Supabase (PostgreSQL).

-> URL BASE LOCAL: http://localhost/Estadias/backend/api
   (Nota: Si tu alias en XAMPP es diferente, ej. /creaciones%20antigravity/Estadias, ajusta la base).
-> HEADERS GLOBALES REQUERIDOS:
   Content-Type: application/json
   Authorization: Bearer <TU_TOKEN_JWT> (Para endpoints protegidos)
================================================================================

--------------------------------------------------------------------------------
1. MÓDULO DE USUARIOS Y ROLES (Supabase Tables: roles, usuario)
--------------------------------------------------------------------------------

[1.1] ROLES (roles)

GET    /roles
URL: http://localhost/Estadias/backend/api/roles
Descripción: Obtener lista de todos los roles.

POST   /roles
URL: http://localhost/Estadias/backend/api/roles
JSON Body:
{
  "nombre": "Soporte Técnico",
  "descripcion": "Gestión de mantenimiento de equipos",
  "permisos": "{\"mantenimiento\":\"all\", \"activos\":\"read\"}"
}

PUT    /roles/{id}
URL: http://localhost/Estadias/backend/api/roles/1
JSON Body:
{
  "nombre": "Administrador General",
  "descripcion": "Control total del sistema (Actualizado)"
}

DELETE /roles/{id}
URL: http://localhost/Estadias/backend/api/roles/3


[1.2] USUARIOS (usuario)

GET    /usuarios
URL: http://localhost/Estadias/backend/api/usuarios

POST   /usuarios
URL: http://localhost/Estadias/backend/api/usuarios
JSON Body (Nota: Mapea al UsuarioController con PostgreSQL):
{
  "nombre": "Juan Perez",
  "correo": "juan.perez@sigrat.edu",
  "contrasena": "Password123!",
  "rol_id": 2,
  "tag_id": "E200001"
}

PUT    /usuarios/{id}
URL: http://localhost/Estadias/backend/api/usuarios/1
JSON Body:
{
  "nombre": "Juan Perez Editado",
  "estatus": "Inactivo",
  "rol_id": 2
}

DELETE /usuarios/{id}
URL: http://localhost/Estadias/backend/api/usuarios/1
Descripción: Baja lógica del usuario (soft delete, actualiza estatus='Inactivo').

4. MÓDULO DE RESERVAS Y PRESTAMOS
--------------------------------------------------------------------------------

[4.1] RESERVAS (reserva)

GET    /reservations
URL: http://localhost/Estadias/backend/api/reservations?esp_id=1&date=2024-12-25
Descripción: Consulta disponibilidad pública.

POST   /reservations
URL: http://localhost/Estadias/backend/api/reservations
JSON Body:
{
  "esp_id": 1,
  "fecha_uso": "2024-12-25",
  "hora_ent": "10:00:00",
  "hora_sal": "12:00:00",
  "motivo": "Clase de Redes Avanzadas"
}

POST   /reservations/bulk
URL: http://localhost/Estadias/backend/api/reservations
JSON Body (Soporta múltiples fechas):
{
  "esp_id": 1,
  "fechas_uso": ["2024-12-25", "2024-12-26"],
  "hora_ent": "10:00:00",
  "hora_sal": "12:00:00",
  "motivo": "Congreso"
}

POST   /reservations/{id}/approve
URL: http://localhost/Estadias/backend/api/reservations/1/approve
Descripción: Aprobación de reserva (Requiere token de Administrador).

POST   /reservations/{id}/reject
URL: http://localhost/Estadias/backend/api/reservations/1/reject
JSON Body:
{
  "reason": "Mantenimiento del aula."
}
